## Security Policy

Last updated: March 4, 2026

### Supported Versions
Security fixes are applied to the latest code on `main`.

### Reporting a Vulnerability
Please report vulnerabilities privately through GitHub repository security
advisories or by contacting the repository owner account directly.

When reporting, include:
- A clear description of the issue
- Steps to reproduce
- Impact and possible attack scenario
- Suggested fix (if available)

### Response Expectations
- Initial acknowledgment target: within 7 days
- Status updates: as available after triage
- Fix timing: based on severity and maintainability

### Scope
This policy covers:
- Static site source code
- Client-side JavaScript behavior
- Hosted deployment behavior directly controlled by this repository

Third-party services and dependencies are handled under their own policies.
